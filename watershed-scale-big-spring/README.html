README

TIN-Based Real-Time Integrated Basin Simulator basin-scale benchmark

This repository hosts the setup for executing a basin-scale tRIBS simulation of the Big Spring watershed, a forested tributary to Sycamore Creek in Northern Arizona, USA.

Obtaining tRIBS

tRIBS v5.2 (or later) uses CMake as build system, instructions on downloading and building tRIBS can be found here. Or alternatively one may use the tRIBS docker image, more information on this can be found here.

Model Execution

From the command line tRIBS can be executed as follows, assuming the executable is stored in the sub-directory bin:

bin/tRIBS <path/to/infile>

Alternatively, if you want to run tRIBS in parallel, you will need to install OpenMPI or a similar Message Passing Interface, and build the parallel version of tRIBS as described here. In order to run the parallel version of tRIBS, you need to call mpirun or similar and specify the number of cores required with the flag -n. The following is an example of this assuming partRIBS is the parallel build of tRIBS.

mpirun -n 3 bin/partRIBS <path/to/parallel_infile>

The Big Spring serial input file can be found in src/in_files/big_spring.in and parallel file can be found in src/in_files/big_spring_par.in. Note: the current input file is setup with relative paths, any modification in the directories structure may also require an update of the .in file.

Benchmark

The benchmark case results are stored as a zip file under results/reference and include sub-directories for both parallel and serial model runs. These results can be visualized using the jupyter notebook located in doc/notebooks/Results.ipynb. In order to run the jupyter notebook we recommend creating a virtual environment using python where you can install additional but required packages as specified in doc/notebooks/requirements.text. Note you may also have to install jupyter in your virtual environment using pip install jupyterlab.

Directory Structure:

data

Contains all the necessary data to run tRIBS at Big Spring and includes graph files for parallel operations. Though additional graph files can be generated using MeshBuilder.

doc

Contains Jupyter notebooks for running and analyzing this specific benchmark case, along side additional documentation.

src

Is designed to contain source code for for the tRIBS executable, which can be obtained here.

bin

Directory for building and storing tRIBS executable, with instructions here.

results

Directory for results from Big Spring tRIBS simulation. The sub-directory references contains additional sub-directories for both parallel and serial model runs.
